webpack и его преимущества

- Первая его функция он минизирует написанный css  код
- Мы может использовать препрожессор SASS - Scss для удобство написания css
Браузер не понимает SASS файлы, для этого нам надо его изменить в читаемый для браузера вид тут на приходит на помощь Node.js.
Окей, как работать. Есть главный SASS файл в котором мы пишем стили, писать мы конечно же может в разных файлах,но подключать будет в главный например main.sass. Дальше выполняеться функции в js файлах таким оброзом что они переписывают файл в нормальный вид для браузера в файле style.css и мы получаем готовый вариант для папки public.

в паппке resources есть папки css и js  в этих папка и будет писаться стили и скрипты, у них есть основыне файлы app к которым надо  подключать остальные файлы.

Выполняем команды в node.js commad prompt
Обязательно надо указать путь к рабочей папке:
Сначала нужно установить засимости из некторые моудли могут зависить от других поэтому они могуть за собой сказачать еще другие модули  package.json:
npm install  - устанавливает много пакетов и создает папку node_modules и package.lock.json
npm run dev - (режим разработчика, удобно читать код) Создает папки в public > css > app.css и js > app.js копируя их из resources
npm run prod - (режим пользователя) Минимализирует файлы, готовые файлы для хостинга
node -v    - показывает версию
npm run watch - команда компилирует файлы сразу при их изменении, это команда висит активным режимом , и не дает не чего написать больше в консоле, из нее надо выходить Ctrl + C выйти из watch режима

При подлючение файлов к главное страницы в laravel принято писатть функции asset('/css/app.css') так и для js И для картинок


Что такое node.js?
Node.js - это менеджер зависимостей для js файлов, такой же как и composer для php
https://www.npmjs.com/ - сайт с пакетами  для js
https://packagist.org/ - сайт с пакетами для php

Все пакетамы устанавливаються в файл package.json в корни проетка
